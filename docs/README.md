# 文書配置

この directory は、planning、product、architecture、利用者向け文書を分けて管理する。

## ディレクトリ

- `product/`: 製品要求、rule、data design、architecture、acceptance criteria、test plan、operations、Lean planning。
- `planning/`: 開発 workflow、リポジトリ構造の決定、modular monolith repository plan。
- `adr/`: 実装決定が固定された後に追加する architecture decision record。
- `user-guides/`: local app workflow が安定した後に追加する利用者向け guide。
