## この文書の位置づけ

この文書は、LaborLens の要求 ID と検証観点を、確認可能な受け入れ基準へ分離するための文書である。

ここで定義するのは、実装が「受け入れ可能」と判断されるための条件である。詳細なテスト手順、テストデータ、性能測定方法、画面レイアウト、CSV 列定義、業務ルールの閾値は、それぞれ `TEST-PLAN.md`、`DATA-DESIGN.md`、`EXTERNAL-DESIGN.md`、`BUSINESS-RULES.md`、`ARCHITECTURE.md` に分離する。

## 受け入れ判定の原則

| 原則 | 内容 |
| --- | --- |
| 要求 ID 追跡 | 各受け入れ基準は、少なくとも 1 つの要求 ID または安全 ID に対応する。 |
| 証跡必須 | 受け入れ判定には、出力ファイル、画面表示、ログ、設定、テスト結果、評価記録などの証跡が必要である。 |
| 原本保護優先 | 原本 CSV の変更、消失、上書きは、他の機能が動作していても受け入れ不可とする。 |
| 安全境界優先 | 個人疲労値、睡眠時間、疲労コメント、少人数部署の健康関連集計、法務・医療・人事評価の断定が出力された場合は受け入れ不可とする。 |
| 未決事項の扱い | 閾値、issue code、性能数値、出力形式などが未決の場合、この文書では「設定・記録・参照・追跡できること」を受け入れ条件にする。具体値は後続文書で確定する。 |
| 推測禁止 | 入力データの粒度、結合可否、法的評価、医療評価、人事評価を推測で確定してはならない。 |

## 優先度

| 優先度 | 意味 | リリース判定 |
| --- | --- | --- |
| P0 | 原本保護、プライバシー、安全境界、判断代替防止に関わる基準 | すべて合格が必要 |
| P1 | CSV 取り込み、検査、照合、粒度判定、レポート、再確認などの主要機能 | 原則として合格が必要 |
| P2 | デモ、評価改善、詳細運用、未決事項に依存する基準 | 未合格の場合は制約として明記する |

## 共通の受け入れ証跡

| 証跡 | 内容 |
| --- | --- |
| `run_summary.json` | `RunId`、対象期間、入力参照、入力ハッシュ、処理状態、issue 件数、成果物参照、実行設定を確認できること。 |
| `issues.csv` | issue 種別、行、列、理由、優先度、関連 `RunId` または入力ハッシュを確認できること。 |
| レポート出力 | データ準備状況、勤怠確認、人件費確認、抑制済み集計、再確認結果などを確認できること。 |
| 実行ログ | ジョブ開始、進捗、完了、失敗理由、再実行に必要な情報を確認できること。 |
| 設定記録 | 少人数部署閾値、労務確認閾値、対象期間、入力データ種別、読み込み設定などを確認できること。 |
| AI 回答評価記録 | 根拠文書表示、安全境界遵守、抑制対象情報への回答拒否、日本語・英語回答品質を確認できること。 |
