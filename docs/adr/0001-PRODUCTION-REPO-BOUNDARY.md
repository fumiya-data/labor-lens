# ADR 0001: 本番 LaborLens を試作品ワークスペースから分離する

Date: 2026-06-01
Status: accepted for planning

## 背景

既存の `labor-lens-prototype` ワークスペースには、Go と Python による
試作品、製品化用パッケージ、生成済みレポート、配布成果物、提案資料が
含まれている。本番アプリケーションは、Rust を中核にしたローカルサーバー、
ローカルDB、バックグラウンドジョブ、ローカルUI を持つ製品として計画している。

両方の取り組みを 1 つのツリーに入れると、異なる目的が混ざってしまう。

- 試作品としての速度と証拠集め
- 本番アーキテクチャ、型安全性、保守性
- 初期試作品の配布パッケージ出力
- 将来の長期運用を前提にした製品コード

## 決定

本番用のリポジトリを次の場所に分離して作成する。

```text
C:\Users\kinbo\labor-lens
```

試作品リポジトリは証拠の source として残す。本番 Rust アプリケーションの
コードを置く場所にはしない。

## 結果

- 本番コードは、最初からクリーンな Rust ワークスペース構成を使える。
- 試作品の出力は、製品要件、テスト、fixture、受入条件へ選択的に変換できる。
- 生成済みの試作品配布ファイルが、本番履歴を汚さない。
- 共有するサンプルデータは、プライバシーと fixture としての目的を文書化した上で、意図的にコピーする必要がある。

## 今後の判断事項

- Git をすぐ初期化するか、最初の Rust/local-server scaffold 後に初期化するか。
- 最初のコード milestone を CLI-first にするか、local server and storage contract-first にするか。
- Python によるレポート rendering をこのリポジトリ外に残すか、将来 controlled helper package として取り込むか。
